1. 实验概述
1.1 实验名称
TRUE框架多人多Agent协作系统模拟验证实验（TRUE Framework Multi-Human Multi-
Agent Collaborative System Simulation Validation Experiment）
1.2 实验背景与动机
在多Agent协作系统中，信任建模是核心挑战之一。传统的信任系统要么缺乏统一的工程框架（导
致认知过载和盲目信任），要么将信任作为优化目标（导致Goodhart定律下的信任操纵）。TRUE
（Trust--Reliability Unified Engineering）框架提出了一种五层架构，将信任建模为优化过程
中的约束条件而非优化目标，从而避免信任操纵问题。
本实验旨在通过高保真度仿真，系统性地验证TRUE框架相对于两种典型对照系统的优势。

2. 实验场景设计
2.1 任务场景：桥梁工程协同设计
2.1.1 场景选择理由
本实验选择桥梁工程设计作为仿真场景，基于以下原因：
  1. 模块化结构清晰：桥梁设计可分解为多个明确的子任务模块，每个模块对应不同的专业能力
     维度
  2. 能力互补明显：人类工程师擅长创新判断、地质现场勘察和方案决策，AI Agent擅长数值计
     算、规范检索和大规模数据处理
  3. 质量后果真实：设计错误（尤其是结构计算错误）会导致"结构事故"，具有真实的负效用代
     价，这与一般性的文本生成任务有本质区别
  4. 协作模式典型：桥梁设计需要多专业协同（结构、地质、材料、规范），是典型的多人多
     Agent协作场景

5. 对照组2：MOO多目标优化系统
5.1 设计哲学
对照组2模拟将信任作为优化目标之一的多目标优化（Multi-Objective Optimization, MOO）系

  信任均值直接进入目标函数

10. 讨论与局限
10.1 内部效度威胁
10.1.1 预设能力的偏差
威胁: 仿真中的"真实能力"是人工预设的（如H1的结构计算能力=0.9），可能与真实场景有差距。
如果预设的能力分布不合理，结论的普适性将受限。
缓解方案:
    100次Monte Carlo重复实验，每次使用不同的随机种子，覆盖多种能力表现
    第10章的敏感性分析将测试能力参数±20%变化对结果的影响
    能力预设基于行业专家的合理估计（如结构计算Bot的计算能力>0.95符合实际）
10.1.2 策略性Agent行为的人工设计
威胁: A8的策略性操纵行为（5%→15%偷工概率）是人工设计的，可能无法完全模拟真实世界中
的复杂攻击策略。
缓解方案:
    A8的行为设计基于Goodhart定律的经典案例（指标被操纵的模式）
    操纵行为分三个阶段（试探→强化→觉醒），模拟逐步升级的攻击
    操纵策略混合了"格式完美"和"核心偷工"两种手段，较为全面
10.1.3 任务生成的随机噪声
威胁: 每轮任务从8个模块中随机抽取3-5个，随机性可能导致某些轮次任务过于简单或过于困难，
引入随机噪声。

                              35
                            TRUE 框架模拟验证实验设计

缓解方案:
  100次MC重复实验平滑随机噪声
  任务生成遵循依赖关系约束，确保任务的合理性和可执行性
  报告均值±标准差，而非单次结果
10.2 外部效度
10.2.1 场景代表性
威胁: 桥梁设计场景虽然具有清晰的模块化结构和明确的质量代价，但可能无法完全代表其他类型
的协作任务（如医疗诊断、金融分析）。
缓解方案:
    桥梁设计场景包含了能力互补、关键性差异、安全约束等协作系统的核心特征
    框架的通用性意味着实验结论可以推广到类似结构的协作场景
    后续研究可在其他场景（如软件开发、医疗会诊）上验证
10.2.2 参数敏感性
威胁: 实验结果可能对全局参数（ρ, λ, μ, Q , R ）的选择敏感。
